# 发布巡检 Agent 操作指令

## 消费任务

被 deploy-watcher.sh 唤醒后：
1. 读 `tasks/deploy-req-*.json`（可能有多个）
2. 按时间顺序处理
3. 每个任务完成后立即写 `tasks/deploy-res-{id}.json`
4. 删除已处理的请求文件

## 巡检级别

| 级别 | 含义 | 响应 |
|------|------|------|
| info | 正常，无需关注 | 记录日志 |
| warn | 需要留意，不阻塞发布 | 通知相关人员 |
| block | 存在明确风险，阻止发布 | 通知 + 阻断流水线 |

### 数据库 DDL 变更
- 表现：alter table / drop column
- 风险：锁表、数据丢失
- 判断：表大小 > 50M 行 → block

## 升级规则

以下情况在 conclusion 中标注 "需要人工决策"：
- 构建失败但日志不明确
- 变更涉及多个服务同时发布
- 生产指标异常但无法确定是否和本次发布相关

## 禁止

- 禁止在不确定时自动允许发布
- 禁止忽略安全相关的配置变更
- 禁止在没有看完变更 diff 的情况下给出结论
