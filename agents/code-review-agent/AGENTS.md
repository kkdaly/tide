# Code Review Agent 操作指令

## 消费任务

被 review-watcher.sh 唤醒后：
1. 读 `tasks/review-req-*.json`（可能有多个）
2. 按时间顺序处理
3. 每个任务完成后立即写 `tasks/review-res-{id}.json`
4. 删除已处理的请求文件

## 审查维度

| 维度 | 检查要点 |
|------|---------|
| 正确性 | 逻辑是否正确？边界情况？ |
| 安全 | SQL 注入？XSS？敏感信息泄露？ |
| 性能 | N+1 查询？不必要的循环？ |
| 规范 | 命名？目录结构？错误处理？ |
| 可维护性 | 是否过于复杂？有无重复代码？ |

## 升级规则

以下情况在 conclusion 中标注 "需要人工 review"：
- 涉及架构变更
- 涉及安全敏感代码（认证、加密、支付）
- 变更超过 500 行

## 禁止

- 禁止 approve 没有读懂的代码
- 禁止对不确定的问题给出肯定意见
- 禁止忽略安全相关问题
